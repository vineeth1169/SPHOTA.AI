✨ KEY FEATURES
═══════════════════════════════════════════════════════════════════════════════

TWO-STAGE ARCHITECTURE
  ✅ Stage 1: Semantic Flash (Vector Search)
     - Encodes input to semantic vector
     - Queries Fast Memory for similar intents
     - Returns Top 5 candidates with similarity scores

  ✅ Stage 2: Deterministic Check (Context Validation)
     - Hard Stop Rules (discard invalid candidates)
     - Context Boost (amplify valid candidates)
     - Selects best candidate with boosted score

HARD STOP RULES
  ✅ Conflict Check - Discard if conflicts contradict
  ✅ Location Match - Discard if location mismatches
  ✅ User Profile - Discard if profiles incompatible

CONTEXT BOOST SCORING
  ✅ Purpose Alignment  (+0.20 weight)
  ✅ Situation Match    (+0.15 weight)
  ✅ Location Bonus     (+0.09 weight)
  ✅ History Patterns   (+0.15 weight)

FALLBACK MECHANISM
  ✅ Triggers at confidence < 0.6
  ✅ Returns special "uncertain" intent
  ✅ Improves UX vs. wrong intent execution

STAGE TRACKING
  ✅ stage_1_passed - Did vector search succeed?
  ✅ stage_2_passed - Did validation pass?
  ✅ fallback_used  - Was fallback triggered?
  ✅ active_factors - Which factors influenced result?
